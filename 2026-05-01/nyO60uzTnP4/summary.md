# Making $$ with AI Agents：从 Howie Lou 这期访谈里读懂 AI Agent 的真正机会

- 视频：`Making $$ with AI Agents`
- YouTube：<https://youtu.be/nyO60uzTnP4>
- 时长：1:05:11
- 嘉宾：Howie Lou（Airtable 联合创始人兼 CEO）

## 先说结论

如果你现在还把 AI 当成一个“更聪明的聊天机器人”，那你大概率还停留在第一幕。Howie Lou 在这期访谈里讲得很直接：**AI Agent 的市场不只是万亿美元，最终很可能对应的是整块白领劳动市场**。换句话说，这不是一个“提高一点效率”的工具周期，而是一个**工作流重构、组织形态重构，甚至创业方式重构**的时代窗口。

这期内容最有价值的地方，不是又喊了一遍“AI 很重要”，而是给出了三个非常具体的判断：

1. **今天的 Agent 能力已经跨过可用门槛**，不再只是演示玩具。
2. **真正的机会不在“聊天增强”，而在“自主执行 + 多工具协同 + 持续记忆迭代”**。
3. **未来最强的个人和团队，不是更会写 prompt 的人，而是更会构建 Agent 系统的人。**

下面我用更适合中文读者的方式，把这期访谈拆开讲清楚。

## 一、为什么说 Agent 的机会远不止“万亿美元”

访谈一开始，主持人引用了 Sequoia 的一个判断：AI Agent 领域有一万亿美元的机会。Howie 的反应很有意思——他几乎是立刻把这个数字继续往上抬。

他的逻辑很简单：

- 如果今天最前沿的 Agent 已经能在软件开发中完成若干小时甚至数天的人类工作；
- 如果模型智能已经足够强，能在多轮上下文、复杂工具调用和长任务中保持一致性；
- 那么 Agent 的目标市场就不应该只按“软件工具市场”算，
- 而应该按**被替代、增强或重构的白领劳动总量**来算。

这就是为什么他说，真正的 TAM（总可服务市场）可能不是一万亿，而是**几十万亿美元量级**。

这个判断并不夸张。很多人一看到 Agent，会本能地把它理解成“Copilot 的升级版”。但 Copilot 是协作式增强，Agent 更接近**任务承包者**：给它目标、上下文和工具，它就能自己研究、执行、迭代、交付。

差别就在这里：

- **Copilot 时代**：人还是主流程，AI 提供建议。
- **Agent 时代**：人设定目标，AI 负责跑流程。

这是从“辅助劳动”到“数字劳动”的切换，市场规模当然不是一个级别。

---

## 二、很多行业还没有真正进入 Agent 时代

访谈里提到一个分布图：当前 AI Agent 的部署主要集中在软件工程领域，接近一半；而销售、营销、后台运营等其它职能占比都不高。

Howie 的判断是：**这恰恰说明渗透率还远远不够**。

他甚至认为，软件工程的高占比本身都可能是一种“错觉”。因为很多企业看起来在用 AI，实际上只是在用第一代增强式工作流：

- 代码补全
- IDE 内嵌助手
- 局部问答
- 人类驱动下的碎片化协作

而前沿团队已经切到了另一种模式：

- 多个云端 Agent 并行工作
- 每个 Agent 连接浏览器、代码环境和工具集
- 自动创建 PR
- 自动互相 review
- 自动收集反馈后继续修改

这不是“一个工程师用了 AI”，而是**软件生产方式被改写了**。

更重要的是，这种模式并不只适用于工程。Howie 的核心观点是：

> 只要模型的智能已经到位，几乎每一个白领岗位都应该被 Agent 重新做一遍。

这包括：

- 市场研究
- 内容创作
- 客户支持
- 销售运营
- 行政与后台流程
- 管理咨询式分析
- 创业中的机会研究与 MVP 搭建

所以现在不是“哪些领域能用 Agent”，而是“**哪些领域还没来得及被 Agent 彻底重构**”。

## 三、很多人低估了 Agent，因为他们根本没认真用过

这段是我最认同的。

Howie 说了一句非常狠、但非常对的话：**using is believing**。很多人不是不相信 Agent，而是他们根本没有完成一次足够深入的使用。

他们做的通常是：

- 丢一个简单问题进去；
- 让 AI 生成一点文字；
- 看看效果还行；
- 然后得出结论：哦，挺有用，但也就那样。

问题是，**这根本不是 Agent 的正确打开方式**。

真正能感受到质变的使用方式，是把一个足够有野心、足够真实、足够长链路的任务交给它，例如：

- 调研一个细分市场并输出商业分析
- 帮你做竞品研究并形成定位建议
- 从公开数据里挖出用户痛点
- 直接搭一个 MVP
- 生成营销内容，再根据反馈迭代风格
- 建一个持续运行、持续学习的业务代理人

当任务开始包含：

1. 多步骤推理
2. 多工具调用
3. 结果验证
4. 反复修订
5. 长时间执行

你才会意识到，自己面对的已经不是一个“回答器”，而是一个**具备执行链条的数字同事**。

这也解释了为什么很多人错失窗口：他们不是判断错了趋势，而是用错了入口。

## 四、HyperAgent 想做的不是另一个 Agent，而是 Agent 的“Mac”

在访谈下半场，Howie 展示了 HyperAgent，并给了一个很聪明的定位：

> 如果别的 Agent 产品更像 Linux，HyperAgent 想做的是 Mac。

这不是在比谁更“极客”，而是在回答一个非常现实的问题：

**Agent 到底什么时候会从少数高阶玩家的玩具，变成普通创业者和团队的日常工作台？**

他的回答是：要靠产品化。

HyperAgent 的几个关键词非常清晰：

- 云原生
- 默认安全
- 开箱即用
- 可视化操作
- 强 UX
- 具备完整任务线程与工具集

说白了，它想解决的是“会的人玩得飞起，不会的人连门都进不去”的断层问题。

这和 Airtable 当年做的事情其实一脉相承：

- 过去做应用需要懂数据层、逻辑层、视图层；
- Airtable 把这些抽象成更直观的 no-code 体验；
- 现在构建 Agent 也很复杂；
- HyperAgent 想把 Agent 的能力封装成更自然、更好上手的系统。

这个方向我认为是对的。技术突破当然重要，但真正能放大市场规模的，往往不是“能力再强一点”，而是**让更多普通人第一次真的用起来**。

---

## 五、Agent 的杀手级价值：它不只是写代码，它能像创始人一样工作

Howie 演示了一个很有代表性的案例：围绕一个创业点子，Agent 不只是“给建议”，而是完成了一整条链路：

1. 研究市场机会
2. 分析竞争格局
3. 收集用户声音
4. 验证真实需求
5. 生成商业论证
6. 直接搭建产品的 V1

这段最关键的点在于：**App building 本身已经开始商品化，真正稀缺的是“上下文整合 + 业务判断 + 从研究到执行的闭环”。**

也就是说，未来强的 Agent 平台，不只是“帮你生成一个应用”，而是：

- 先理解你为什么要做这个应用；
- 再理解市场、用户和竞争关系；
- 然后再做一个更有业务针对性的版本；
- 之后还能继续运营、监控、改进。

Howie 用一句话概括得很好：

> HyperAgent 在这个场景里，不只是开发者，更像创始人。

这句话有点营销，但方向没错。真正值钱的，不是“把代码写出来”，而是**把业务目标拆成一个可执行系统，并持续推进**。

## 六、技能（Skills）和记忆（Memory）才是 Agent 系统的底层复利

如果说模型能力决定了 Agent 的上限，那技能和记忆决定了它的复利速度。

访谈里，Howie 多次强调 skills 的重要性。可以把它理解为：

- 一套可复用的工作方法
- 一种对任务的结构化操作说明
- 一个能被多次调用并持续改进的执行模块

比如一个内容 Agent，不该每次都从零开始思考“怎么写一条像 Greg 风格的推文”。它应该拥有：

- 对目标平台的理解
- 对作者口吻的偏好
- 对好内容标准的 rubric
- 对过去反馈的记忆
- 对改写方向的持续学习

这就不再是一次性的 prompt engineering，而是**工作系统工程**。

同样，记忆也不是“把聊天记录存起来”这么简单。真正有价值的记忆是：

- 哪些输出效果好
- 哪些风格更受欢迎
- 哪些错误经常发生
- 哪些评估标准是稳定有效的
- 哪些技能模块值得固化

这就像给 Agent 建立 SOP、质检规则和组织知识库。

所以我非常认同一个判断：**未来最强的 AI 使用者，不是最会写 prompt 的人，而是最会训练、组织、评价和迭代 Agent 的人。**

## 七、对个体创业者来说，最现实的机会是什么？

访谈里还有一条很实用的创业判断：Howie 很喜欢那种**不是百亿级超级大市场，但足够做出高利润业务的中型市场**。

这是一个很成熟的创始人视角。

很多人做 AI 创业时，一上来就盯着“下一个 OpenAI”“下一个 Salesforce”。这当然很燃，但对个人和小团队来说，通常也是最容易被巨头碾死的路线。

Howie 更看好的，是那些：

- 市场规模有几亿到几十亿美元
- 用户问题真实存在
- 大公司嫌它不够大
- 但小团队用 Agent 可以极高效率切入

这类市场的特点是：

1. **足够具体**，容易做深；
2. **不需要堆特别多人**，Agent 能显著放大单兵效率；
3. **可以从研究、验证、开发到运营一路自动化加速**；
4. **能在细分场景里建立很高的利润密度**。

这其实是 AI Agent 给普通创业者最公平的礼物：

> 它不一定让你一夜之间打赢所有大公司，但它大幅提高了你在细分战场上独立作战的胜率。

## 八、Agent 不是偶尔用一下，而是要像练肌肉一样每天用

访谈后半段聊到一个很现实的问题：普通人怎样才能真正成为“前 1% 的 Agent Builder”？

答案没有任何神秘感：**高频、持续、制度化使用。**

Greg 提到一个很好的习惯：把使用 Agent 的时间直接塞进日历里，每天 30 分钟、60 分钟，固定练。

这背后有两个原因：

### 1. Agent 的使用收益是复利型的

你今天学会一个技能模块，明天能复用；
你今天积累一个 workflow，下周还能自动跑；
你今天写出一套评估标准，未来每次内容输出都能套上去。

会用的人，能力不是线性增长，而是系统叠加。

### 2. 认知切换需要时间

你很难靠一次周末冲刺，突然变成 Agent 时代的高手。更现实的路径是：

- 先把它用于一个小任务；
- 再让它承担一个真实工作流；
- 再开始把记忆、技能、评估引进来；
- 最后形成自己的 Agent 团队与操作系统。

这和写作、健身、编程都很像：

**不是灵感驱动，而是习惯驱动。**

## 九、我从这期访谈里提炼出的 5 个行动建议

如果你不是来听趋势鸡血，而是想知道“那我明天该做什么”，我会给出这 5 条：

### 1. 不要只问问题，开始交付任务

停止把 AI 只当搜索框或写作助手。给它一个完整任务：调研、分析、生成、验证、交付。

### 2. 用 Agent 做你最耗时、最重复、最需要多步骤的工作

这类工作最容易被重构，也是最能让你立刻看到 ROI 的地方。

### 3. 别盯最大市场，先盯最可打穿的细分市场

Agent 时代，小团队的优势不是规模，而是速度、深度和成本结构。

### 4. 早点建立自己的 skills、memory、rubric

别每次重新写 prompt。把有效方法沉淀成模块，把好坏标准显性化。

### 5. 给自己 30 天连续使用实验

每天固定时间，拿真实业务场景练。不是“有空试试”，而是把它当成工作系统升级计划。

---

## 最后：别把 Agent 当工具升级，要把它当组织升级

这期访谈真正让我警醒的一点是：**很多人仍在用“工具替换”的视角看 Agent，但真正发生的事情是“组织能力替换”。**

当一个 Agent 能：

- 理解目标
- 调用工具
- 维护上下文
- 接受反馈
- 迭代技能
- 长期运行

它就不再只是一个软件功能，而是一个可以嵌进团队里的数字岗位。

所以接下来最重要的问题，不是“AI 会不会有用”，而是：

> 你会不会用 Agent 重写你的工作方式、创业方式，甚至团队结构？

Howie Lou 这期访谈的核心价值，就在这里：它把“Agent 很厉害”这句空话，推进成了一个更务实也更残酷的判断——

**窗口已经开了，真正落后的不是模型，而是人的工作流。**

如果你还没开始系统地用 Agent，现在就是补课时间。

而且老实说，再拖就有点蠢了。🦞
